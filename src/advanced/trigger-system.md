# 触发器系统

触发器系统分为触发器与执行器两块内容，一个触发器可以对应触发多个执行器，同一个执行器也可以被多个触发器触发。

## 触发器 `0_Extras → Functional → Trigger`

触发器可以通过玩家球触发，或通过指定物体触发，也可以被执行器强制触发。触发器必须关联执行器，否则没有任何意义。

### `triggerType`

- 类型：`enum{Enter, Stay, Exit}`
- 默认值：`Enter`

触发器类型。`Enter` 在进入触发器区域时使能；`Stay` 在停留于触发器中时按 `triggerCoolDown` 的设定反复使能；`Exit` 在离开触发器区域时使能。

### `executors`

- 类型：`Tuple<int, Executor[]>`
- 默认值：`(0, [])`
- 限制：`(n, l): n >= 0`

执行器列表。触发器使能时，这些触发器将**同时**开始执行。

### `triggerObjectIds`

- 类型：`Tuple<int, int[]>`
- 默认值：`(0, [])`
- 限制：`(n, l): n >= 0 && forall i in l {i > 0 || i in specialList}`

有效元件 ID 列表。只有列表中对应的物体可以使能触发器。以下为具有特殊意义的 ID：

- `-99`：玩家球
- `-100`：玩家木球
- `-101`：玩家石球
- `-102`：玩家纸球
- `-103`：玩家冰球
- `-104`：玩家钢球
- `-105`：玩家橡胶球
- `-106`：玩家光球
- `-107`：玩家气球
- `-108`：玩家粘球
- `-109`：玩家海绵球
- `-200`：玩家球以外的所有刚体

### `triggerWorkableCount`

- 类型：`int`
- 默认值：`-1`
- 限制：`n: n >= -1`

可使能次数。`-1` 表示无限制。

### `triggerCoolDown`

- 类型：`float`
- 默认值：`0`
- 限制：`t: t >= 0`

冷却时间。特别地，对于 `Stay` 类型的触发器，当 `triggerCoolDown == 0` 时，触发器将在**每个物理帧**使能一次。

## 执行器 `0_Extras → Functional → Executor`

执行器可以被触发器触发，也可以关联执行器实现链式触发。

::: tip

执行器的位姿和缩放没有作用，也不会写入 `.bms` 文件。不过建议摆放在便于查找的位置。

:::

### `M++ Script`

- 类型：`string`
- 默认值：`""`

执行器 M++ 脚本。详见 [M++ `Executor` 文档](/mpp/executor)。
